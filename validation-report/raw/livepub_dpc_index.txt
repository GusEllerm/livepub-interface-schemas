<h2>Vocabulary Terms</h2>

<h2 id="HardwareRuntime">dpc:HardwareRuntime</h2>
<p>
  A snapshot of the hardware and environment configuration for an executed distributed step.
  Bundles together all hardware components that together describe the runtime environment 
  (node, VM, container host) in which the step ran.
</p>

<h3>Domain / Usage</h3>
<ul>
  <li>Type: <code>rdfs:Class</code></li>
  <li>Contains: <code>dpc:HardwareComponent</code> instances via <code>dpc:component</code></li>
  <li>Appears in Distributed Provenance Crates to capture hardware characteristics at runtime.</li>
</ul>

<h2 id="HardwareComponent">dpc:HardwareComponent</h2>
<p>
  A physical or virtual hardware element (e.g., CPU, GPU, RAM, storage volume, network interface) 
  captured at the time a workflow step executed.
</p>

<h3>Domain / Usage</h3>
<ul>
  <li>Type: <code>rdfs:Class</code></li>
  <li>Used in: <code>dpc:HardwareRuntime</code> via <code>dpc:component</code></li>
  <li>May link to: <code>schema:Observation</code> via <code>dpc:performance</code> for metrics</li>
  <li>Appears in Distributed Provenance Crates to describe individual hardware components.</li>
</ul>

<h2 id="component">dpc:component</h2>
<p>
  Links a <code>dpc:HardwareRuntime</code> to each <code>dpc:HardwareComponent</code> that was observed.
  This property relates the runtime snapshot to its constituent hardware components.
</p>

<h3>Domain / Usage</h3>
<ul>
  <li>Type: <code>rdf:Property</code></li>
  <li>Domain: <code>dpc:HardwareRuntime</code></li>
  <li>Range: <code>dpc:HardwareComponent</code></li>
</ul>

<h2 id="performance">dpc:performance</h2>
<p>
  Links a <code>dpc:HardwareComponent</code> to an <code>schema:Observation</code> that reports 
  metrics like CPU load, memory usage, I/O throughput, network bandwidth, etc.
</p>

<h3>Domain / Usage</h3>
<ul>
  <li>Type: <code>rdf:Property</code></li>
  <li>Domain: <code>dpc:HardwareComponent</code></li>
  <li>Range: <code>schema:Observation</code></li>
</ul>
